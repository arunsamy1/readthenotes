Here’s a step-by-step guide to clean your boot partition and remove Windows entirely from a Windows + Ubuntu dual-boot system.

⚠️ Before you start

  • Back up your data – This process deletes Windows completely.
  • Have a live Ubuntu USB handy (in case you need to repair GRUB).

Step 1: Backup Windows data (optional)

Boot into Windows and copy any files you want to keep to an external drive or Ubuntu partition.

Step 2: Boot into Ubuntu

Make sure you boot into your Ubuntu installation, not the live USB.

Step 3: Remove Windows from the boot menu

Open a terminal and run:

sudo update-grub

This regenerates the GRUB menu. Ubuntu will still list Windows until Windows is actually removed from the disk, but this ensures GRUB is refreshed later.

Step 4: Delete the Windows partition(s)

  1. Open GParted (install if missing: sudo apt install gparted).
  2. Identify your Windows partitions:

- Usually labeled NTFS or fat32 (system reserved)

- The main Windows partition is large (C: drive)

  1. Right-click each Windows partition → Delete.
  2. Apply changes (Edit → Apply All Operations).
💡 If Ubuntu is using the same EFI partition as Windows, don't delete the EFI partition – only delete the NTFS/FAT32 partitions that belong to Windows.

Step 5: Reclaim space (optional)

After deleting Windows partitions, you can:

  • Create a new partition for data
  • Extend your Ubuntu partition into the free space (in GParted → resize/move)

Step 6: Clean up boot entries (UEFI systems)

If you are on a UEFI system (most modern PCs):

# List EFI boot entries
sudo efibootmgr -v

Find the Windows Boot Manager entry and delete it:

sudo efibootmgr -b XXXX -B

(Replace XXXX with the boot number for Windows)

Then remove the Windows bootloader files:

sudo mount /dev/nvme0n1p1 /mnt   # Mount your EFI partition (check device name with lsblk)
sudo rm -rf /mnt/EFI/Microsoft
sudo umount /mnt
The EFI partition is usually small (~100–500 MB) and formatted as FAT32. Do not delete the EFI/ubuntu folder.

Step 7: Final GRUB update

sudo update-grub

Now GRUB should show only Ubuntu.

Step 8: Verify

Reboot. The system should boot directly into Ubuntu with no Windows option in GRUB.

If something goes wrong

Boot from the Ubuntu live USB, then:

sudo mount /dev/sdXY /mnt          # Your Ubuntu root partition
sudo mount /dev/sdXZ /mnt/boot/efi # Your EFI partition
sudo grub-install --boot-directory=/mnt/boot /dev/sdX
sudo update-grub

Replace sdXY, sdXZ, and sdX with your actual devices.

Summary (TL;DR)

  1. Boot Ubuntu
  2. Use GParted → delete Windows partitions
  3. (UEFI) Use efibootmgr + remove EFI/Microsoft
  4. Run sudo update-grub
  5. Reboot – Ubuntu only ✅
